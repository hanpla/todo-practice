# Stage 1 힌트 — 기본 CRUD

---

## 1. 데이터 불러오기 (Stage1.jsx)
- JSON 파일은 `import` 문으로 바로 불러올 수 있습니다.
- React에서 변하는 데이터를 관리하려면 어떤 Hook을 사용하나요?
- Hook의 인자로 불러온 데이터를 전달하면 초기 상태가 됩니다.

## 2. 할 일 추가 (TodoForm.jsx → Stage1.jsx)
- `<form>`의 `onSubmit` 이벤트를 활용하세요.
- 폼 제출 시 기본 동작(페이지 새로고침)을 막으려면 어떤 메서드를 호출해야 하나요?
- 새 todo 객체를 만들 때 고유한 `id`가 필요합니다. `Date.now()`를 활용해보세요.
- 기존 배열에 새 항목을 추가할 때, 원본 배열을 직접 수정하면 안 됩니다.
  - 스프레드 연산자(`...`)를 활용하여 새 배열을 만들어보세요.

## 3. 완료 토글 (TodoItem.jsx → Stage1.jsx)
- `<input type="checkbox" />`의 `checked`와 `onChange`를 활용하세요.
- 배열에서 특정 id를 가진 항목만 수정하려면 `map`을 사용합니다.
- 객체의 특정 속성만 변경할 때도 스프레드 연산자를 활용할 수 있습니다.

## 4. 삭제 (TodoItem.jsx → Stage1.jsx)
- 배열에서 특정 항목을 제거할 때 유용한 배열 메서드가 있습니다.
- 조건에 맞는 항목만 남기는 메서드를 찾아보세요.

## 5. 이름 변경 (TodoItem.jsx)
- 더블클릭(`onDoubleClick`) 이벤트로 수정 모드에 진입합니다.
- 수정 모드일 때와 아닐 때 다른 요소를 보여주려면 조건부 렌더링을 사용하세요.
- `onKeyDown` 이벤트에서 `e.key`로 어떤 키가 눌렸는지 확인할 수 있습니다.
- 입력창에서 포커스가 벗어날 때(`onBlur`)도 수정을 확정하면 사용자 경험이 좋습니다.

## 6. 컴포넌트 간 데이터 흐름
- 상태(state)는 **부모 컴포넌트(Stage1.jsx)**에서 관리합니다.
- 자식 컴포넌트에는 **props**로 데이터와 함수를 전달합니다.
- `onAdd`, `onToggle`, `onDelete`, `onRename` — 이런 함수들을 props로 내려주세요.
